## RentalStatus

#ADMIN_CANCELLED = "ADMIN_CANCELLED" - новый статус, реализуй

Нет ADMIN_CANCELLED статуса:
В AdminRentalService._CANCEL_STATUS_MAP_ админская отмена маскируется под REJECTED_BY_OWNER / CANCELLED_BY_OWNER. 
Это значит, что по данным невозможно отличить «владелец отменил» от «админ отменил» без проверки audit log. 
Стоит добавить ADMIN_CANCELLED в RentalStatus.


Там, где rental переводится в terminal cancel-status:
при установке одного из статусов:
    REJECTED_BY_OWNER
    CANCELLED_BY_OWNER
    CANCELLED_CONFIRMED_BY_OWNER
    ADMIN_CANCELLED
нужно проставлять:
cancelled_at=datetime.now(timezone.utc)


Там, где rental переводится в COMPLETED:
нужно проставлять:
completed_at=datetime.now(timezone.utc)


было
что-то в духе:
CANCEL_STATUS_MAP = {
    ...: RentalStatus.REJECTED_BY_OWNER,
    ...: RentalStatus.CANCELLED_BY_OWNER,
}

должно стать
для админской отмены:
RentalStatus.ADMIN_CANCELLED
