### Future: Unit of Work / transaction middleware

Сейчас repository write-method сам владеет `commit/rollback`. Это допустимо для MVP, но в будущем нужно вынести transaction boundary на уровень `UnitOfWork` или transaction middleware.

Это позволит:
- убрать rollback из каждого repository;
- выполнять несколько repository-операций в одной атомарной транзакции;
- централизованно управлять commit/rollback;
- сделать services чище и надёжнее.

После внедрения UoW repositories не должны делать `commit()` и `rollback()` — только работать с ORM/session.
