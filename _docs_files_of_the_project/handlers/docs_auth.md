# EditProfileHandlers

[**Router-файл для FSM-редактирования профиля пользователя**]()

> `process_edit_name/email`: получаем пользователя из БД вручную (миддлваре здесь НЕ работает ???)

>Файл отвечает за сценарии редактирования пользовательского профиля: 
> - показ меню редактирования
> - запрос нового имени
> - сохранение имени
> - запрос нового email
> - сохранение email

### Методы:

> `show_edit_profile_settings` - **[Показывает подменю редактирования профиля]()**

> `ask_new_name` - **[Запрашивает новое имя пользователя]()**
>
> - переводит FSM в: `ProfileEditStates.waiting_for_name`
> - показывает ввод имени

> `process_edit_name` - **[Обрабатывает ввод нового имени пользователя]()**
>
> - получает новое имя
> - валидирует имя 
> - если update успешен: очищает FSM
>    - показывает success-текст
>    - показывает кнопку возврата в профиль


> `ask_new_email` - **[Запрашивает новый email пользователя]()**
> 
> - переводит FSM в: `ProfileEditStates.waiting_for_email`
> - показывает ввод email 

> `process_edit_email` - **[Обрабатывает ввод нового email пользователя]()**
>
> - получает новый email
> - валидирует email
> - если update успешен:
>    - очищает FSM
>    - показывает success-текст
>    - показывает кнопку возврата в профиль

# PhoneHandlers

[**Router-файл для смены номера телефона в профиле пользователя**]()

Файл отвечает только за сценарий смены телефона уже зарегистрированного пользователя: 
 - запрос нового контакта
 - проверку принадлежности контакта текущему Telegram-пользователю
 - обновление телефона через `UserService` и возврат в профиль

> * `tg_user` — это Telegram-пользователь (Это всё данные из Telegram, не из БД). Но этот пользователь НЕ хранится в твоей базе
> * `user` — это SQLAlchemy-модель из middleware (Это твоя база данных)

> Обрабатывает контакт-телефон:
>    - ✔ Регистрация нового пользователя - УБРАЛ
>    - ✔ Смена номера

> `await state.clear()` чтобы FSM после регистрации гарантированно завершался

### Методы:

> `request_phone_number_change` - **[Запрашивает новый номер телефона для смены в профиле]()**
> 
> - переводит FSM в: `ProfileEditStates.waiting_for_phone`
> - сценарий смены номера

> `process_phone_number` - **[Обрабатывает новый контакт-телефон пользователя]()**
