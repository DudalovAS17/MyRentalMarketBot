# BaseHandlers

[**Router-файл базовых entrypoint-сценариев и главного меню**]()

> - `/start` и `🏠 Главное меню`
> - `show_main_menu_callback = menu:main` / `back_to_main_menu`
> - `text_message_handler`
> - `/help` - `/legal` - `/cancel` - `/ (unknown_command)` - `noop`

### Методы:

### Start and ShowMainMenu

> `start` - **[Обрабатывает `/start` и вход через кнопку главного меню]()**
>
> - очищает FSM через `state.clear()`
>    - 🧹 очищаем старое состояние, чтобы бот всегда начинает “с чистого листа” (если пользователь когда-то застрял в регистрации)
> - получает `Telegram ID` пользователя
> - [если action `REGISTER`:]() - новый пользователь = регистрируем
>     - запускает `start_registration(...)`
>     - Логика "Если нет телефона" - отдельно не обрабатывается
> - [если action `ACCESS_BLOCKED`:]() - Проверяем блокировку
>     - показывает сообщение о блокировке
> - [иначе:]()
>     - показывает главное меню через `show_main_menu(...)`


> `show_main_menu_callback` - **[Показывает главное меню по callback-кнопке]()**
>
> - срабатывает на: `menu:main` / `back_to_main_menu`
> - вызывает `show_main_menu(callback, user)`
>
> Здесь не нужно проходить через `/start`, как для `message`, так как 
> там передается аргумент `user` (то есть регистрация уже пройдена)

### Commands

> `help_command` - **[Показывает помощь]()**
> * Принимает: `message`
> * Внутри: отправляет `HELP_TEXT`
> * Срабатывает на: `/help`

> `legal_command` - **[Показывает юридическую информацию]()**
> * Принимает: `message`
> * Внутри: отправляет `LEGAL_TEXT`
> * Срабатывает на: `/legal`

> `cancel` - **[Отменяет текущий сценарий и возвращает в главное меню]()**
> * Принимает: `message`
> * Срабатывает на: `/cancel`
> * FSM очищается полностью
> 
> [TODO: если отменено создание объявления - сообщение пользователю: черновик сохранен для будущего использования]()
>
> - очищает FSM через `state.clear()` - удаляет и состояние, и все данные FSM.
>   - Очищаем временные данные/ключи - `for_search`, `item_data`, `search_filters`, `search_city` и тп
> - показывает текст отмены
> - возвращает пользователя в главное меню

> `unknown_command` - **[Отвечает на неизвестную slash-команду]()**
> * Принимает: `message`
> * Срабатывает на: любой текст, начинающийся с `/`, если его не обработали выше
>
> Внутри:
>    - получает исходную команду
>    - формирует текст
>    - отправляет ответ с main-menu keyboard

> `noop` - **[Закрывает callback-заглушку]()**
> * Принимает: `callback`
> * Срабатывает на: callback `noop`
>
> Внутри: отвечает `Недоступно`

### Text Message Handler

> `text_message_handler` - **[Обрабатывает текстовые кнопки главного меню]()**
> * Принимает: `message`
> * определяет по тексту (или по ID кнопки), что пользователь хотел
> * вызывает нужный сценарий (handler-функцию)
>
> Внутри:
>    - нормализует текст
>    - определяет действие
>    - если действие найдено: вызывает action
