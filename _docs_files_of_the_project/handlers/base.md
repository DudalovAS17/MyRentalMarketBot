# Base — базовые входы и главное меню

Документ описывает актуальный слой `handlers/base/*` для текущего проекта: Telegram-бот компании по аренде товаров. Базовый роутер не содержит бизнес-логики аренды, каталога или поддержки — он только определяет стартовую точку, общие команды и маршрутизацию текстовых кнопок главного меню.

---

## 1. Base

| Команда                     | Обработчик          | Поведение                                                                                                                                                                                                        |
|-----------------------------|---------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `/start`                    | `start()`           | Универсальная точка входа, кнопка **[🏠 Главное меню]()**                                                                                                                                                        |
| `/help`                     | `help_command()`    | Отправляет `HELP_TEXT` - помощь                                                                                                                                                                                  |
| `/legal`                    | `legal_command()`   | Отправляет `LEGAL_TEXT` - юридическая информация                                                                                                                                                                 |
| `/cancel`                   | `cancel()`          | Очищает FSM, сообщает об отмене и возвращает главное меню. **[Является глобальным пользовательским способом выйти из сценария]()**. Он не откатывает уже созданные записи в БД - только сбрасывает FSM-контекст. |
| неизвестная `/...`          | `unknown_command()` | Отвечает на неизвестную slash-команду. Отвечает текстом `build_unknown_command_text(command)`.                                                                                                                   |
| `noop`                      | `noop()`            | Закрывает callback-заглушку: отвечает коротким сообщением «Недоступно».  Используется для кнопок-заглушек, заголовков или временно отключённых действий.                                                         |
| `show_main_menu_callback()` |                     | Ниже                                                                                                                                                                                                             |
| `text_message_handler()`    |                     | Ниже                                                                                                                                                                                                             |

`base_router` подключается последним в `app/routers.py`, потому что содержит общий обработчик `@base_router.message(F.text)`. Если подключить его раньше, он начнёт перехватывать текстовые сообщения, которые должны попасть в FSM других разделов.

## 2. Base helpers

| Функция                    | Поведение                                            |
|----------------------------|------------------------------------------------------|
| `safe_answer_for_blocked`  | Сообщить пользователю, что его аккаунт заблокирован  |
| `normalize_menu_text`      | Нормализовать текст кнопки главного меню             |
| `resolve_main_menu_action` | Определяет действие по тексту кнопки главного меню   |

---

## 3. `start()` — универсальная точка входа

Обработчик срабатывает на:
- `/start`;
- текстовую кнопку `🏠 Главное меню`.

Алгоритм:
1. очищает текущий FSM через `state.clear()`;
2. получает Telegram ID пользователя;
3. вызывает `user_service.resolve_start_entry(tg_id)`;
4. действует по `StartAction`:
   - `REGISTER` — создаёт первичную запись пользователя и просит контакт;
   - `NEED_PHONE` — повторно просит отправить контакт;
   - `ACCESS_BLOCKED` — показывает сообщение о блокировке;
   - иначе показывает главное меню.

Важно: регистрация в проекте минимальная. Пользователь создаётся по Telegram-данным, но до подтверждения телефона не считается полностью готовым к работе с ботом.

---

## 4. Callback главного меню

`show_main_menu_callback()` обрабатывает callback-кнопки:
- `menu:main`;
- `back_to_main_menu`.

Он подтверждает callback через `callback.answer()` и вызывает общую entry-функцию `show_main_menu(callback, user)`. 
Пользователь (`user`) приходит из middleware, поэтому сам обработчик не ходит в БД.

---

## 5. Catch-all для текстовых кнопок

`text_message_handler()` обрабатывает любой обычный текст, который не был перехвачен более конкретными роутерами.

Обрабатывает текстовые кнопки главного меню.

Алгоритм:
1. нормализует текст через `normalize_menu_text()`;
2. вызывает `resolve_main_menu_action(...)`;
3. если action не найден — отвечает `UNKNOWN_MAIN_MENU_TEXT`;
4. если action найден — выполняет соответствующую entry-функцию.

Текущие основные действия меню:

- открыть категории каталога;
- открыть «Мои заявки»;
- открыть поддержку;
- открыть профиль/настройки через профильные хендлеры;
- вернуться в главное меню.

---

## 6. Правила разработки

1. Новые FSM-сценарии подключать в своих роутерах раньше `base_router`.
2. В `base_router` не добавлять бизнес-операции: создание аренды, смену статуса, модерацию товара и т.п.
3. Новые текстовые кнопки главного меню добавлять через `helpers_base.resolve_main_menu_action()`, а не через отдельный catch-all.
4. Любой сценарий, который начинается из главного меню, должен иметь отдельную entry-функцию или специализированный роутер.
5. При изменении `/start` учитывать `StartAction`: нельзя показывать каталог пользователю без завершённой регистрации или заблокированному аккаунту.
