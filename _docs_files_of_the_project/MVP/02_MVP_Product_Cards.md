# MVP: Карточки товаров

## 1. Назначение блока

Карточка товара — это главный продающий экран бота. Пользователь принимает решение оставить заявку именно здесь.

Для RMB карточка должна быть похожа на мини-страницу товара:

```text
фото
название
цена
наличие
характеристики
описание
кнопка заявки
кнопка вопроса
```

---

## 2. Что уже есть в проекте

В проекте уже есть:

```text
краткая карточка в карусели
подробная карточка товара
вывод цены
вывод наличия
вывод минимального срока аренды
вывод характеристик
фото товара
кнопка “Подробнее”
кнопка “Оставить заявку”
кнопка “Написать менеджеру” / вопрос
карусель товаров с предыдущим/следующим товаром
```

Это хороший фундамент. Карточки не нужно делать с нуля.

---

## 3. MVP-минимум карточки

Карточка товара для MVP должна показывать:

1. Главное фото.
2. Название товара.
3. Цену аренды.
4. Минимальный срок аренды.
5. Наличие или понятный статус.
6. 3–6 ключевых характеристик.
7. Краткое описание.
8. Кнопку заявки.
9. Кнопку вопроса по товару.
10. Навигацию назад.

Минимальный формат:

```text
🏗 Виброплита Champion PC9045F

💰 Цена: от 1100 ₽/сутки
📦 В наличии: 2 шт.
⏱ Минимальный срок: 1 сутки

Характеристики:
• Вес: 91 кг
• Мощность: 5,5 л.с.
• Глубина уплотнения: до 200 мм

Описание:
Подходит для уплотнения грунта, песка и щебня.
```

---

## 4. Кнопки карточки

Минимальные кнопки:

```text
🛒 Оставить заявку
💬 Задать вопрос
⬅️ Назад
```

Для карусели товаров:

```text
⬅️ Предыдущий
1/9
➡️ Следующий
ℹ️ Подробнее
🛒 Оставить заявку
```

Лучше, чтобы “Оставить заявку” была доступна уже из карусели, а не только после “Подробнее”. Это сокращает путь клиента.

---

## 5. Фото

Для MVP допустим один из двух вариантов:

### Вариант A — простой

```text
Главное фото в карточке
+ кнопка “Показать все фото”
```

Если сейчас в проекте уже так работает, это можно оставить для MVP.

### Вариант B — лучше

```text
⬅️ Фото
1/6
➡️ Фото
```

Фото-карусель лучше для UX, но не обязательна для первичного MVP.

---

## 6. Что нужно слегка поправить

### 6.1. CTA из карусели - ПОКА НЕ РЕАЛИЗУЕМ!

Добавить кнопку заявки прямо в карусельную карточку, если её там нет.

Цель:

```text
пользователь увидел товар
→ сразу может оставить заявку
```

### 6.2. Статус товара

Сейчас может показываться только наличие. Для MVP лучше показать понятный клиенту статус:

```text
✅ В наличии
🟡 Наличие уточняет менеджер
⛔ Сейчас недоступно
```

Технический enum можно не показывать напрямую.

### 6.3. Вопрос по товару - ПОКА НЕ РЕАЛИЗУЕМ!

Кнопка “Задать вопрос” должна создавать support ticket с `item_id`.

Если сейчас кнопка есть, но не привязывает вопрос к товару — это нужно довести.

---

## 7. Что НЕ входит в MVP карточек

Пока не делать:

```text
отзывы в карточке
рейтинг товара
похожие товары
избранное
сложные рекомендации
сравнение товаров
полную фотогалерею уровня сайта
```

Отзывы можно оставить как кнопку/заглушку, если уже есть, но не развивать до завершения заявки.

---

## 8. Критерии готовности

Карточки готовы для MVP, если:

```text
пользователь понимает:
что это за товар
сколько стоит аренда
есть ли товар
какие основные характеристики
как оставить заявку
как задать вопрос
как вернуться назад
```

---

## 9. Приоритет

Статус: почти готово.

Приоритет доработки: средний.

Главная доработка — не дизайн, а сокращение пути до заявки и правильная связь вопроса с товаром.
