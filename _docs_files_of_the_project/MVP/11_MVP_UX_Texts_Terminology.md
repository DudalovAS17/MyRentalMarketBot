# MVP: Тексты и терминология

## 1. Назначение блока

Тексты и терминология нужны, чтобы весь бот говорил на языке одного продукта.

RentalMarketBot — это не C2C-маркетплейс и не доска объявлений. Это Telegram-система заявок для компании по аренде оборудования.

Поэтому язык интерфейса должен поддерживать именно эту модель.

---

## 2. Что уже есть в проекте

В проекте уже есть отдельные `texts/` и много готовых сообщений. Но по отчёту видно, что местами сохраняются старые термины:

```text
сделки
объявления
создать объявление
модерация объявлений
```

Для нового RMB это нужно заменить.

---

## 3. Единый словарь MVP

Использовать:

| Правильный термин | Смысл |
|---|---|
| товар / оборудование | единица каталога компании |
| каталог аренды | основной раздел товаров |
| заявка | запрос клиента на аренду |
| менеджер | сотрудник компании |
| админ-панель | рабочее место сотрудника |
| поддержка | обращение клиента |
| клиент | обычный пользователь |

Не использовать в клиентском и админском UX:

| Убрать | Заменить на |
|---|---|
| объявление | товар |
| создать объявление | добавить товар |
| модерация объявлений | управление товарами |
| сделка | заявка |
| арендодатель | компания / менеджер |
| владелец товара | компания |

---

## 4. MVP-минимум текстов

Нужно привести к единому стилю:

```text
главное меню
каталог
карточка товара
поиск
заявка
успешное создание заявки
мои заявки
админская карточка заявки
поддержка
уведомления клиенту
уведомления менеджеру
пустые состояния
ошибки
```

---

## 5. Тон интерфейса

Тон должен быть:

```text
короткий
деловой
понятный
без лишней “игровой” стилистики
```

Пример хорошего текста:

```text
Заявка отправлена менеджеру.
Мы уточним наличие оборудования и свяжемся с вами.
```

Пример плохого текста:

```text
Сделка создана! Ждите ответа арендодателя.
```

---

## 6. Что нужно подправить

1. Пройтись по всем текстам и кнопкам.
2. Заменить C2C-термины на B2C-термины аренды оборудования.
3. Разделить клиентские и админские формулировки.
4. Проверить, что заявка везде называется заявкой.
5. Проверить, что товар везде называется товаром/оборудованием.
6. Убрать неоднозначные тексты вроде “сделка”, если речь о заявке.

---

## 7. Что НЕ входит в MVP

Пока не делать:

```text
мультиязычность
сложную tone-of-voice систему
редактирование текстов из админки
A/B тесты текстов
```

---

## 8. Критерии готовности

Блок готов, если:

```text
пользователь нигде не видит “объявления” и “сделки”
админ работает с “товарами” и “заявками”
все основные экраны звучат как один продукт
сообщения короткие и понятные
```
