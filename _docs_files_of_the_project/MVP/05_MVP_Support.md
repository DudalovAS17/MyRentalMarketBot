# MVP: Поддержка

## 1. Назначение блока

Поддержка нужна, чтобы клиент мог задать вопрос до или после заявки.

В аренде оборудования вопросы неизбежны:

```text
есть ли доставка?
подойдёт ли товар?
можно ли забрать сегодня?
какой срок?
есть ли в наличии?
какой залог?
что с моей заявкой?
```

Поддержка должна быть связана с контекстом: товаром или заявкой.

---

## 2. Что уже есть в проекте

Уже есть:

```text
SupportTicket
статусы OPEN / CLOSED
связь тикета с user_id
связь с item_id
связь с rental_id
создание тикета через /support или кнопку
создание тикета из карточки заявки
уведомление админов о новом тикете
админский список тикетов
карточка тикета
ответ админа
закрытие тикета
уведомление клиента об ответе
уведомление клиента о закрытии
```

Это достаточно хороший MVP-фундамент.

---

## 3. MVP-минимум поддержки

Поддержка должна позволять:

1. Создать обращение из главного меню.
2. Создать вопрос по товару.
3. Создать вопрос по заявке.
4. Показать обращение админу.
5. Ответить клиенту.
6. Закрыть обращение.
7. Уведомить клиента и менеджера.

---

## 4. Типы обращений

Минимально нужны 3 контекста:

```text
Общий вопрос
Вопрос по товару
Вопрос по заявке
```

Связи:

```text
general ticket:
user_id

item ticket:
user_id + item_id

rental ticket:
user_id + rental_id
```

---

## 5. Что нужно слегка поправить

### 5.1. Вопрос по товару

Если кнопка “Задать вопрос” в карточке товара уже есть, она должна создавать тикет с `item_id`.

Минимальный flow:

```text
💬 Задать вопрос
→ пользователь пишет вопрос
→ создаётся SupportTicket(item_id=...)
→ админ видит, по какому товару вопрос
```

### 5.2. История сообщений

Сейчас SupportTicket может работать как “один вопрос — один ответ”. Для минимального MVP это допустимо.

Но рабочий вариант лучше строить так:

```text
SupportTicket
SupportMessage
```

Минимальная модель `SupportMessage`:

```text
ticket_id
sender_type: user / admin / system
sender_user_id
sender_admin_id
text
created_at
```

Не обязательно делать сложный чат сразу. Достаточно сохранять первое сообщение пользователя и ответ админа как сообщения.

### 5.3. Карточка тикета

Админ должен видеть:

```text
номер тикета
статус
клиент
телефон / username
связанный товар, если есть
связанную заявку, если есть
текст вопроса
последний ответ
дата создания
```

Кнопки:

```text
💬 Ответить
✅ Закрыть
⬅️ Назад
```

---

## 6. Что НЕ входит в MVP поддержки

Пока не делать:

```text
полноценный чат с бесконечной перепиской
назначение тикетов на менеджеров
SLA
приоритеты тикетов
файлы/вложения
шаблоны быстрых ответов
оценку качества поддержки
```

Это можно добавить позже.

---

## 7. Критерии готовности

Поддержка готова для MVP, если:

```text
клиент может задать вопрос
вопрос сохраняется
админ получает уведомление
админ видит контекст вопроса
админ может ответить
клиент получает ответ
админ может закрыть тикет
```

Дополнительный критерий:

```text
вопрос по товару содержит item_id
вопрос по заявке содержит rental_id
```
