# MVP: Каталог

## 1. Назначение блока

Каталог — это входная точка в продукт. Его задача: быстро привести клиента от общего раздела оборудования к конкретному товару, по которому можно оставить заявку.

**[Категория → Подкатегория → Товары → Карточка → Заявка]()**

---

## 2. Что уже есть в проекте

В текущем проекте каталог уже является одним из самых зрелых контуров.

Уже есть:
- Category
- Subcategory
- Item
- ItemPhoto
- ItemCharacteristic
- seed/import-данные
- фото товаров в storage
- карусель товаров
- навигация назад по каталогу

Клиентский путь:
- главное меню
- каталог
- категории
- подкатегории
- товары подкатегории

---

## 3. MVP-минимум каталога

Для MVP каталог должен уметь:

1. Показывать только активные категории.
2. Показывать подкатегории внутри выбранной категории.
3. Показывать товары внутри выбранной подкатегории.
4. Не показывать скрытые/архивные товары клиенту.
5. Уметь возвращать пользователя назад:
   - к подкатегориям;
   - к категориям;
   - в главное меню.
6. Использовать seed/import-данные как базовое наполнение каталога.

---

## 4. Минимальная структура данных

Каталог для MVP должен опираться на текущие сущности:

```text
Category
- id
- name
- parent_id
- is_active
- sort_order

Item
- id
- category_id
- subcategory_id
- title/name
- description
- price
- available_quantity
- status
- rental_period

ItemPhoto
- item_id
- telegram_file_id
- url
- is_main
- sort_order

ItemCharacteristic
- item_id
- name
- value
- sort_order
```

Важно: если в текущем проекте поля называются немного иначе, не нужно переименовывать их ради красоты. Главное — сохранить смысл.

---

## 5. Что нужно слегка поправить

### 5.1. Терминология

Убрать из клиентского каталога C2C-термины:

| Было               | Должно быть         |
|--------------------|---------------------|
| объявления         | товары              |
| сделки             | заявки              |
| арендодатель       | компания / менеджер |
| создать объявление | добавить товар      |


### 5.2. Фильтрация товаров

В клиентском каталоге должны показываться только товары, которые можно показать клиенту.

Минимальное правило:

```text
item.status == ACTIVE
и item.available_quantity > 0
```

### 5.3. Сортировка

Для MVP достаточно сортировать:

```text
sort_order ASC
name/title ASC
```

Сложные фильтры не нужны.

---

## 6. Что НЕ входит в MVP каталога

Пока не делать:

```text
сложные фильтры
избранное
рекомендации
персональные подборки
корзину
несколько товаров в одной заявке
полноценную веб/Mini App-витрину
```

Поиск уже есть в проекте — его можно оставить, но он не должен мешать доводке основного пути через каталог.

---

## 7. Критерии готовности

Каталог можно считать готовым для MVP, если:

```text
пользователь может открыть каталог
→ выбрать категорию
→ выбрать подкатегорию
→ увидеть товары
→ перейти в карточку товара
→ вернуться назад без потери контекста
```

Дополнительный критерий:

```text
в каталоге нет терминов “объявления”, “сделки”, “арендодатель”
```
Это пока не касается файла `texts/base.py` - на него пока "закрываем глаза".
