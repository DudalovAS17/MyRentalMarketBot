# MVP: Управление товарами админом

## 1. Назначение блока

Этот блок нужен, чтобы менеджер или админ мог управлять витриной оборудования без прямого вмешательства в базу.

Но для MVP важно не пытаться построить полноценную CMS. Каталог уже может жить на seed/import-данных. Поэтому админское управление товарами должно быть минимальным.

---

## 2. Что уже есть в проекте

В проекте уже есть элементы шире MVP:

```text
админские разделы товаров
“Создать объявление”
“Модерация объявлений”
статусы товаров DRAFT / ACTIVE / HIDDEN / ARCHIVED
админская смена статуса товара
модели товара, фото, характеристик
seed/import-данные
```

Главная проблема сейчас не отсутствие логики, а терминология и приоритеты.

---

## 3. MVP-минимум

Для MVP достаточно, чтобы админ мог:

1. Открыть список товаров.
2. Открыть карточку товара в админке.
3. Скрыть товар от клиентов.
4. Вернуть товар в каталог.
5. Изменить наличие/количество.
6. Изменить цену, если это уже легко поддерживается.
7. Видеть статус товара.

---

## 4. Минимальные действия

```text
📦 Товары
🔎 Найти товар
👁 Открыть товар
✅ Опубликовать
🙈 Скрыть
🗄 Архивировать
📦 Изменить наличие
💰 Изменить цену
```

Если создание товара уже есть и работает — оставить. Но не ставить его выше доработки заявок.

---

## 5. Терминология

Заменить:

| Старое | Новое |
|---|---|
| Создать объявление | Добавить товар |
| Модерация объявлений | Управление товарами |
| Объявление | Товар |
| Автор объявления | Админ / компания |

Пользователь не создаёт товар. Товаром управляет только компания.

---

## 6. Что можно оставить на seed/import

Для MVP допустимо не делать полное создание товара через Telegram-админку.

Можно оставить через seed/import:

```text
массовое добавление товаров
загрузку фото
создание характеристик
создание категорий
```

А в админке дать только управление видимостью, ценой и наличием.

---

## 7. Что НЕ входит в MVP

Пока не делать:

```text
полноценный редактор галереи
сложный редактор характеристик
массовое редактирование
импорт Excel через бота
модерацию пользовательских товаров
публикацию товаров пользователями
```

---

## 8. Критерии готовности

Блок готов, если:

```text
админ может открыть товар
→ понять его статус
→ скрыть/показать товар
→ изменить наличие
→ при необходимости изменить цену
→ клиентский каталог корректно учитывает эти изменения
```

---

## 9. Приоритет

Статус: частично есть, но шире MVP.

Приоритет: средний.

Сначала довести заявки и админку заявок. Потом аккуратно привести управление товарами к новой B2C-терминологии.
