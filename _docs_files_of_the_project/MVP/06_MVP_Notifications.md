# MVP: Уведомления

## 1. Назначение блока

Уведомления — это то, что превращает бот из каталога в рабочий канал продаж.

Главная логика:

```text
клиент оставил заявку
→ менеджер сразу получил уведомление
→ менеджер обработал
→ клиент получил статус
```

Если уведомления работают плохо, бизнес-ценность бота резко падает.

---

## 2. Что уже есть в проекте

Уже есть `NotificationService` и покрытие основных событий:

```text
уведомление админов о новой заявке
уведомление клиента о создании заявки
уведомление клиента об изменении статуса
уведомление админов об отмене заявки клиентом
уведомление админов о новом тикете поддержки
уведомление клиента о создании тикета
уведомление клиента об ответе поддержки
уведомление клиента о закрытии тикета
```

Это сильная сторона текущего проекта.

---

## 3. MVP-минимум уведомлений

Минимально нужны уведомления по 3 группам:

### 3.1. Заявки

```text
новая заявка → админам/менеджерам
заявка создана → клиенту
заявка взята в работу → клиенту, опционально
заявка подтверждена → клиенту
заявка отклонена → клиенту
заявка отменена → клиенту/админам
заявка завершена → клиенту
```

### 3.2. Поддержка

```text
новый тикет → админам
тикет создан → клиенту
ответ админа → клиенту
тикет закрыт → клиенту
```

### 3.3. Системные ошибки

Для MVP достаточно логировать ошибки. Отдельные уведомления owner/admin о критических ошибках можно добавить позже.

---

## 4. Уведомление менеджеру о новой заявке

Сообщение должно быть структурированным:

```text
🆕 Новая заявка #42

Товар: ...
Количество: ...
Срок: ...
Доставка: ...
Адрес: ...

Клиент: ...
Телефон: ...
Telegram: ...

Комментарий:
...

Статус: новая
```

Кнопки:

```text
✅ Взять в работу
✅ Подтвердить
❌ Отклонить
📞 Контакт клиента
📝 Комментарий
```

---

## 5. Уведомление клиенту

Тон должен быть спокойный и понятный.

Пример после создания заявки:

```text
✅ Заявка отправлена

Мы передали её менеджеру.
Он проверит наличие, срок аренды и доставку.

Номер заявки: #42
Статус: новая
```

Пример после подтверждения:

```text
✅ Заявка #42 подтверждена

Менеджер подтвердил возможность аренды.
Если нужно, он свяжется с вами для уточнения деталей.
```

Пример после отклонения:

```text
❌ Заявка #42 отклонена

Причина:
...

Вы можете выбрать другой товар или написать менеджеру.
```

---

## 6. Что нужно слегка поправить

### 6.1. Actionable-кнопки

Уведомление менеджеру должно не просто информировать, а давать действие:

```text
открыть заявку
взять в работу
подтвердить
отклонить
```

### 6.2. Единый стиль текстов

Все уведомления должны быть в одном стиле:

```text
короткий заголовок
номер сущности
статус
что произошло
что делать дальше
```

### 6.3. Защита от падений

Уведомление не должно ломать бизнес-операцию.

Пример:

```text
заявка создана в БД
→ уведомление админу не отправилось
→ заявка всё равно должна существовать
→ ошибка логируется
```

---

## 7. Что НЕ входит в MVP уведомлений

Пока не делать:

```text
email-уведомления
SMS
WhatsApp
массовые рассылки
маркетинговые цепочки
сложные retry-очереди
отложенные уведомления
```

Для MVP достаточно Telegram-уведомлений и логирования ошибок.

---

## 8. Критерии готовности

Уведомления готовы для MVP, если:

```text
после создания заявки менеджер получает сообщение
после создания заявки клиент получает подтверждение
после смены статуса клиент получает сообщение
после тикета поддержки админ получает сообщение
после ответа поддержки клиент получает сообщение
ошибка отправки не ломает создание заявки
```
