# MVP: Поиск оборудования

## 1. Назначение блока

Поиск нужен, когда клиент уже примерно знает, что ему нужно: “виброплита”, “генератор”, “перфоратор”, “бетономешалка”.

Поиск не должен заменять каталог. Он должен быть быстрым входом в карточку товара.

```text
Поиск → результаты → карточка товара → заявка
```

---

## 2. Что уже есть в проекте

В текущем проекте поиск уже существует как расширение:

```text
search_router
команда /search
FSM для ввода поискового запроса
пагинация результатов
поиск товаров по названию/описанию
переход к товару
```

Это хороший задел. Для MVP не нужно строить сложный фильтратор.

---

## 3. MVP-минимум

Поиск должен уметь:

1. Запускаться из главного меню кнопкой `🔎 Поиск оборудования`.
2. Запускаться командой `/search`.
3. Принимать текстовый запрос.
4. Искать только по активным товарам.
5. Показывать результаты краткими карточками.
6. Давать переход в подробную карточку товара.
7. Показывать понятное пустое состояние, если ничего не найдено.
8. Позволять вернуться в каталог или главное меню.

---

## 4. Правила поиска для MVP

Минимально достаточно искать по:

```text
названию товара
описанию товара
характеристикам — опционально, если уже удобно
```

Не нужно на MVP:

```text
морфология
сложные фильтры
диапазон цен
география
сортировка по популярности
полнотекстовый search engine
```

Если товаров немного, обычный SQL-поиск вполне достаточен.

---

## 5. Что нужно подправить

### 5.1. Добавить поиск в главное меню

Если кнопки ещё нет в пользовательском меню, добавить:

```text
🔎 Поиск оборудования
```

### 5.2. Терминология результатов

Результаты должны называться не “объявления”, а “товары” или “оборудование”.

Пример:

```text
Найдено товаров: 5
Выберите оборудование из списка:
```

### 5.3. Пустое состояние

Если ничего не найдено:

```text
По вашему запросу ничего не найдено.
Попробуйте другое название или откройте каталог.
```

Кнопки:

```text
🏗 Открыть каталог
🔎 Новый поиск
🏠 Главное меню
```

---

## 6. Что НЕ входит в MVP

Пока не делать:

```text
фильтры по цене
фильтры по бренду
фильтры по мощности
поиск по нескольким категориям с фасетами
историю поисков
рекомендации на основе поиска
```

---

## 7. Критерии готовности

Поиск готов, если:

```text
пользователь нажимает “Поиск оборудования”
→ вводит запрос
→ получает список активных товаров
→ открывает карточку
→ может оставить заявку
```

---

## 8. Приоритет

Статус: уже частично реализован.

Приоритет: средний.

Поиск полезен, но не должен идти выше доработки полноценной заявки.
