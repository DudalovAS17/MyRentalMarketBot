
## 1. Оплата 💳

Для аренды строительного оборудования оплата часто не такая простая, как в магазине:
- аренда зависит от срока
- наличия
- доставки
- скидки
- договора
- менеджерского подтверждения

Поэтому для MVP правильно: **[заявка → менеджер уточняет → подтверждение]()**, 
а не сразу “оплатить”.

> **_Оплата:_**
> 
> → Клиент создает заявку
> 
> → Менеджер подтверждает заявку: [бот присылает уведомление с оплатой]()
> (ЮKassa / Telegram Stars / CloudPayments / Robokassa)
>
> → Клиент получает реквизиты/ссылку/инструкцию
> 
> → Менеджер вручную отмечает оплату

Где возможные деньги:
- оплата
- предоплата?
- доставка?
- продление аренды?

## 2. Завершение аренды и отзыв ⭐

Для MVP достаточно:
> → Менеджер нажал “Завершить заявку”
> 
> → статус `COMPLETED`
> 
> → клиент получил уведомление

Позже можно развить:
- дата фактического возврата
- комментарий менеджера
- состояние товара после возврата
- штраф / доплата
- фото возврата
- Отзыв

Отзывы - позже:
- отзыв о товаре
- отзыв о сервисе/компании

## 3. Защита сценариев 🛡️

Вот это очень важный блок.

Что сюда входит:

**[1. Запрет нелогичных действий]()**
- нельзя отменить уже завершённую заявку
- нельзя подтвердить отклонённую заявку
- нельзя взять в работу заявку, которая уже отменена
- нельзя оставить заявку на скрытый товар
- нельзя открыть товар, который удалён/архивирован
- нельзя повторно отправить одну и ту же форму два раза

(должно жить в service/status, не в handler)

**[2. Защита от старых кнопок]()**

_Telegram-боты часто страдают от stale callbacks:_
- пользователь открыл старое сообщение
- нажал старую кнопку
- статус уже изменился
- товар уже скрыт
- заявка уже отменена

Нужно, чтобы бот отвечал спокойно:
"Эта заявка уже изменила статус.
Откройте актуальную карточку заявки."

**[3. Защита от флуда]()**

Минимально:
- ограничить частоту создания заявок
- ограничить частоту сообщений в поддержку
- ограничить поиск слишком частыми запросами
- ограничить повторное нажатие кнопок

Для MVP можно просто:
- не чаще 1 заявки на товар за N минут
- не чаще 1 support ticket за N минут

**[4. Контроль ошибок FSM]()**

_Защита сценариев — обязательна для сильного MVP._

Очень важно:
- пользователь ввёл мусор
- пользователь нажал назад
- пользователь вышел в главное меню
- пользователь начал другой сценарий
- пользователь прислал фото вместо текста
- пользователь нажал старую кнопку внутри FSM

## 4. Репутация усиленная 🏅

**[Для клиента (история клиента для менеджера):]()**
- количество заявок
- количество завершённых аренд
- количество отмен
- признак надёжного клиента
- внутренний комментарий менеджера

Это лучше показывать админу, а не клиенту.

**[Для товара (рейтинг):]()**
- средняя оценка товара
- количество отзывов
- сколько раз арендовали
- популярность

**[Для компании]()**
- отзывы о сервисе
- рейтинг компании
- социальное доверие

## 5. Возврат пользователя 🔁

- повторить заявку
- арендовать снова
- посмотреть завершённые
- скачать/посмотреть детали прошлой аренды
- Быстрый повтор аренды

>Вы уже арендовали:
>Виброплита Wacker Neuson MP-15
> 
>[Арендовать снова]

Это реально полезно для строительной ниши: люди часто берут одно и то же оборудование повторно.

**[Избранные товары]()**
- добавить товар в избранное
- быстро вернуться
- сохранить для будущей аренды

## 6. Рекомендации 🎯

Примеры:
- к виброплите → бензорез / трамбовка
- к генератору → удлинитель / тепловая пушка
- к мойке → шланги / насадки
- к бетономешалке → вибратор / шлифмашина

Для MVP можно проще:
- Похожие товары
- С этим часто арендуют
- Популярное в категории

Это можно сделать даже без “умного AI”:
- по категории
- по тегам
- по ручной связке item_related
- по популярности

## 7. Умные напоминания ⏰

Примеры:
- завтра заканчивается аренда
- нужно вернуть оборудование сегодня
- хотите продлить аренду?
- менеджер ждёт подтверждения
- вы начали заявку, но не отправили

(это требует scheduler/background tasks)

Минимально позже:
- напоминание менеджеру о заявках в статусе REQUESTED дольше N часов
- напоминание клиенту перед окончанием аренды

## 8. Аналитика активности 📊

- сколько пользователей открыли каталог
- какие категории смотрят
- какие товары открывают
- на каких шагах бросают заявку
- какие запросы ищут
- сколько заявок создано
- сколько заявок подтверждено
- сколько отменено

(можно сделать через AnalyticsMiddleware или EventLog)

Но для MVP можно начать проще:
- views_count у товара
- orders_count у товара
- количество заявок по статусам
- популярные категории

## 9. Адаптация под разные ниши 🧩

Бот может стать не просто “ботом аренды строительного оборудования”, а шаблоном:
- аренда инструментов
- аренда спецтехники
- прокат платьев
- аренда фотостудий
- аренда оборудования для мероприятий
- прокат авто/велосипедов/самокатов
- аренда бытовой техники

Но чтобы это получилось, нужно отделить:
- универсальное ядро
- от нишевых текстов/категорий/форм

Универсальное ядро:
- каталог
- карточка товара
- заявка
- админка
- поддержка
- уведомления
- статусы

Нишевые настройки:
- названия категорий
- поля заявки
- условия аренды
- тексты
- доставка/самовывоз
- залог

## 10. Залог и документы 🪪

Для аренды оборудования часто нужны:
- паспорт
- залог
- договор
- реквизиты
- акт передачи

В MVP не надо, но в бизнес-версии можно добавить:
- тип клиента: физлицо/юрлицо
- нужен ли договор
- размер залога
- статус залога
- документы проверены

## 12. Быстрый звонок / контакт менеджера 📞

В карточке товара и заявке:
- 📞 Позвонить
- 💬 Написать менеджеру

Telegram-бот не всегда может открыть звонок красиво, но можно показывать контакт:
- Телефон менеджера
- username
- рабочее время

---

## 13. Рабочее время и автоответы 🕘

Для поддержки и заявок:
- Мы получили заявку.
- Менеджер ответит в рабочее время: 9:00–18:00.

Если заявка ночью:
- Заявка принята. Мы свяжемся с вами утром.

Это сильно повышает доверие.

---

## 14. Черновик заявки 📝

Если пользователь начал заявку, но не закончил:
- выбрал товар
- выбрал срок
- не отправил

Можно позже напомнить или дать продолжить.

Минимально: при выходе из FSM спрашивать: "Вы хотите отменить заявку?"

---

## 15. Админские заметки по клиенту 🗒️

Для менеджера полезно:
- клиент часто арендует
- надёжный
- требует предоплату
- были отмены
- особые условия доставки

Это не показывать клиенту.
