# Правильная ответственность

```text
config.py
    отвечает за настройки

main.py
    отвечает за запуск приложения

app/container.py
    отвечает за сборку services/repositories

app/routers.py
    отвечает за подключение routers

app/middlewares_setup.py
    отвечает за подключение middlewares

db/bootstrap.py
    отвечает за проверку/инициализацию БД
```

# Startup-сценарий в проекте

```text
python main.py
    ↓
asyncio.run(main())
    ↓
configure_logging()
    ↓
settings.log_startup_warnings()
    ↓
build_app()
    ↓
init_db_or_fail()
    ↓
create_bot()
    ↓
create_dispatcher()
    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

```

# Shutdown-сценарий

```text
KeyboardInterrupt / CancelledError / crash
    ↓
finally
    ↓
shutdown_application(bot)
    ↓
close_bot_session(bot)
    ↓
shutdown_db()
```

# Почему такая структура хороша для проекта

Эта структура подходит для `RentalMarketBot`, потому что проект уже разделён на слои:

```text
handlers → services → repositories → models
```

- `main.py` не должен знать детали rental/support/catalog logic.
- `config.py` не должен знать детали Telegram handlers.
Это делает проект проще поддерживать и развивать.

>- `main.py` — точка входа и composition root приложения.
>- `config.py` — единый typed config приложения через `pydantic-settings`.
>
>Цель этих файлов — дать проекту стабильный startup-flow, fail-fast конфиг, централизованную сборку зависимостей и безопасное завершение приложения.
