# Category — каталог, подкатегории, карусель товаров и фото

Раздел описывает актуальные handlers каталога. Каталог принадлежит компании: товары создаются и модерируются сотрудниками, а клиенты просматривают их и создают заявки на аренду.

---

## 1. Карта файлов

| Файл                                               | Назначение                                                                                           |
|----------------------------------------------------|------------------------------------------------------------------------------------------------------|
| `handlers/category/category.py`                    | Основной `category_router`: категории, подкатегории, карусель товаров, детальная карточка, все фото. |
| `handlers/category/category_helpers/keyboard.py`   | Inline-клавиатуры каталога и карточки товара.                                                        |
| `handlers/category/category_helpers/texts.py`      | Тексты карточек, характеристики и пустые состояния.                                                  |
| `handlers/category/category_helpers/load.py`       | Безопасная загрузка сущностей/списков с обработкой ошибок сервиса.                                   |
| `handlers/category/category_helpers/store.py`      | Сохранение выбранной категории, подкатегории и товара в FSM.                                         |
| `handlers/category/category_helpers/formatters.py` | Источник фото, media group и текст занятости товара.                                                 |

## 2. Основной сценарий каталога

1. `show_categories()` из `handlers/entries.py` показывает категории верхнего уровня.
2. `show_subcategories()` показывает подкатегории выбранной категории.
3. `show_items_in_subcategory()` показывает первый товар подкатегории в формате карусели.
4. `navigate_items_carousel()` листает товары.
5. `show_item_details_in_subcategory()` открывает подробную карточку товара.
6. `back_to_categories` - Кнопка '🔙 Назад' → возвращаемся к списку категорий
7. `show_all_photos` - Показать все фотографии товара
8. helper: `send_or_edit_item_card` - Показать карточку товара: с фото, если оно есть, иначе обычным текстом

## 3. Детально

### `show_subcategories()`

Callback начинается с `CAT_CB_PREFIX`.

Обработчик:
- парсит ID категории через `parse_callback()`;
- загружает категорию через `category_service.get_category_by_id()`;
- сохраняет выбранную категорию в FSM;
- загружает подкатегории через `category_service.list_subcategories(category.id)`;
- показывает клавиатуру подкатегорий.

Если категория не найдена или сервис вернул ошибку, пользователю показывается безопасное сообщение.

---

### `show_items_in_subcategory()`

Callback начинается с `SUBCAT_CB_PREFIX`.

Обработчик:
- загружает подкатегорию;
- сохраняет её в FSM;
- вызывает `item_service.list_items_by_subcategory(subcategory.id)`;
- берёт первый товар;
- загружает фото через `photo_service.get_photos_by_item_id()`;
- загружает краткий список характеристик через `item_service.list_item_characteristics_by_item_id(..., limit=3)`;
- показывает карточку через `send_or_edit_item_card()`.

Важный UX: карточка может быть отправлена как фото с caption или как обычный текст, если фото нет.

---

### Карусель товаров

`navigate_items_carousel()` обрабатывает callback с `CAROUSEL_NAV_CB`.

Payload содержит:
- `subcategory_id`;
- индекс товара.

Обработчик повторно загружает подкатегорию и список товаров, нормализует индекс через `% total`, затем перерисовывает карточку текущего товара. Такой подход устойчив к кнопкам «назад/вперёд» и не требует хранить весь список товаров в FSM.

---

### Детальная карточка товара

`show_item_details_in_subcategory()` обрабатывает `ITEM_DETAILS_CB`.

Данные карточки:
- товар из `item_service.get_item_by_id()`;
- названия категории/подкатегории из FSM;
- характеристики товара;
- информация о занятости из `rental_service.get_open_rental_for_item(item.id)`.

Если для товара есть открытая заявка/аренда, клавиатура получает `is_busy=True` и текст `busy_until_text(open_rental)`. Ошибка проверки занятости не блокирует UX: товар всё равно показывается, но без busy-индикатора.

---

### Все фотографии товара

`show_all_photos()` обрабатывает `SHOW_ALL_PHOTOS_CB`.

Алгоритм:
1. парсит `item_id`;
2. загружает фото через `photo_service.get_photos_by_item_id(item_id)`;
3. строит `InputMediaPhoto` через `build_photo_media()`;
4. удаляет предыдущий экран, если Telegram позволяет;
5. отправляет media group;
6. отправляет отдельную кнопку возврата к карточке товара.

Если фото нет, пользователь получает сообщение «У этого товара пока нет доступных фото» и кнопку назад.

---

## 4. FSM-данные каталога

Каталог использует FSM не как бизнес-состояние, а как временный UI-контекст:
- `selected_category_id`;
- `selected_category_name`;
- `selected_subcategory_id`;
- `selected_subcategory_name`;
- `selected_item_id`.

Эти данные нужны для красивой карточки и возврата, но не являются источником прав доступа или цены.

## 6. Правила разработки

1. Не хранить в FSM весь список товаров или фото.
2. Перед арендой всегда повторно загружать товар и проверять доступность в rental flow.
3. Для списка каталога использовать только публичные/активные товары на уровне сервиса.
4. Не блокировать просмотр карточки из-за ошибки проверки занятости — лучше показать карточку без busy-индикатора.
5. Любая новая кнопка карточки должна иметь отдельный callback prefix в `utils/callbacks.py`.
