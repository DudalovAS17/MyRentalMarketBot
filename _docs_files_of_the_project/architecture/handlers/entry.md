# Entries — общие входы

Этот файл описывает общие функции, которые используются несколькими handler-разделами. 
Они не являются самостоятельным роутером, но формируют единый UX: главное меню, категории, список заявок, старт регистрации и тексты уведомлений.

---

## 1. Карта файлов

| Файл                       | Назначение                                                                 |
|----------------------------|----------------------------------------------------------------------------|
| `handlers/entries.py`      | Entry-функции для главного меню, каталога, списка заявок и регистрации.    |
| `handlers/entry_helper.py` | Тексты приветствия и проверка собственного Telegram contact.               |

---

## 2. `handlers/entries.py`

### `show_main_menu(event, user)`

Показывает главное меню клиента. Текст строится через `build_main_menu_text(user)`, клавиатура — через `get_main_menu_keyboard()`.

Функция принимает `Message` или `CallbackQuery`, поэтому может использоваться:

- после `/start`;
- после завершения регистрации;
- после отмены сценария;
- после закрытия пользовательского flow.

### `show_categories(event, category_service)`

Показывает верхний уровень каталога. Функция:

1. подтверждает callback, если вход пришёл из inline-кнопки;
2. вызывает `category_service.list_main_categories()`;
3. при ошибке сервиса показывает безопасное сообщение;
4. отдаёт клавиатуру `build_categories_screen_keyboard(categories)`.

Каталог в текущем проекте — это каталог компании, а не пользовательские объявления.

### `show_my_rentals(event, rental_service, user)`

Показывает список заявок текущего пользователя:

1. вызывает `rental_service.list_rentals_by_user(user.id)`;
2. если заявок нет — показывает пустое состояние и кнопку возврата/каталога;
3. если заявки есть — строит клавиатуру `build_my_rentals_keyboard(rentals, current_user_id=user.id)`.

Здесь не показываются чужие заявки: выборка всегда ограничена `user.id`.

### `start_registration(message, user_service)`

Создаёт первичную запись пользователя по Telegram-профилю:

- `telegram_id`;
- `username`;
- `first_name`;
- `last_name`;
- `full_name`;
- `language_code`;
- `phone=None` до подтверждения контакта.

После создания вызывает `request_phone_confirmation(message)`.

### `request_phone_confirmation(message)`

Отправляет приветственный текст регистрации и reply-кнопку запроса контакта. Контакт должен прийти именно как Telegram contact, а не как произвольный текст.

## 3. `handlers/entry_helper.py`

| Функция                             | Роль                                                                        |
|-------------------------------------|-----------------------------------------------------------------------------|
| `build_main_menu_text(user)`        | Формирует приветствие главного меню с именем пользователя.                  |
| `build_registration_welcome_text()` | Формирует текст первичной регистрации и объясняет, зачем нужен телефон.     |
| `is_own_contact(contact, tg_user)`  | Проверяет, что пользователь отправил собственный контакт, а не чужой номер. |

`is_own_contact()` используется как в регистрации, так и при смене телефона в профиле.

---

## 4. Практические правила

1. Entry-функции должны быть тонкими: собрать данные через сервис и вызвать `send_reply`/`send_or_edit`.
2. Форматирование уведомлений держим отдельно от сервисов и репозиториев.
3. В уведомлениях нельзя раскрывать внутренние технические ошибки пользователю.
4. Все тексты, которые содержат пользовательские данные, должны быть готовы к пустым значениям.
