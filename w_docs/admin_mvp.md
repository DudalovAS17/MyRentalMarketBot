# Admin MVP (1 страница)

## 1) Admin MVP: какие сценарии покрыты

### Модерация сделок
- просматривает и фильтрует сделки по статусам/датам
- подтверждает/отклоняет спорные или проблемные сделки
- фиксирует итоговое решение по спору
- видит ключевые параметры сделки (item, участники, сроки)

### Модерация объявлений
- просматривает новые/обновлённые объявления
- переводит Item из PENDING в ACTIVE или REJECTED
- скрывает/возвращает объявления (ACTIVE↔HIDDEN)
- проверяет корректность контента и соответствие правилам

### Поддержка
- принимает обращения пользователей и классифицирует их
- даёт ответ/решение по обращению
- эскалирует сложные кейсы в модерацию
- закрывает обращение с итоговым статусом

### Управление пользователями
- просматривает карточку пользователя и его историю
- блокирует/разблокирует аккаунт (account_status)
- ограничивает доступ при нарушениях
- фиксирует причину блокировки

## 2) Инварианты/гарантии системы
- Whitelist переходов статусов Item: PENDING→ACTIVE/REJECTED, ACTIVE↔HIDDEN
- Каталог/поиск показывает только Item со статусом ACTIVE
- Open rentals constraint: нельзя скрыть Item (ACTIVE→HIDDEN), если есть активные/открытые сделки по этому item
- BANNED guard: пользователь с account_status=BANNED глобально блокируется middleware (кроме админа)

## 3) Точки расширения (Post-MVP / MVP+)
- список забаненных пользователей, последние зарегистрированные
- расширенная админ-аналитика
- уведомления пользователям о решениях модерации/бане
- отдельная audit-таблица (audit logs) вместо хранения audit-полей в Item/User
- жалобы/FAQ (как опциональные модули, сейчас выключены)
