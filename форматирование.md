
## Новое форматирование

- `<strong>...</strong>` - жирный шрифт
- `<p>...</p>` - абзац
- `<h2>...</h2>` - заголовок большими буквами

### Скрытие абзаца
```python
<details>
  <summary>Показать подробности</summary>
  <p>Тут длинное объяснение...</p>
</details>
```

### Коллаж
```python
<tg-collage>
  <img src="https:/...jpg">
  <img src="https:/...jpg">
</tg-collage>
```

### Slideshow / карусель через стрелочки
```python
<tg-slideshow>
  <img src="https:/...jpg">
  <img src="https:/...jpg">
</tg-slideshow>
```

### Таблица
```python
<table bordered striped>
    <tbody>
        <tr>
        <td><strong>Name</strong></td> - заголовки таблицы
        <td><strong>City</strong></td> - заголовки таблицы
        </tr>
            <tr>
                <td>John</td>
                <td>Chicago</td>
            </tr>
            <tr>
                <td>Lucy</td>
                <td>Wisconsin</td>
            </tr>
            <tr>
                <td>Amanda</td>
                <td>Madison</td>
            </tr>
    </tbody>
</table>
```

# Примеры вывода rich-helper функций Telegram

Документ показывает, какой HTML/объект возвращает каждая helper-функция из предложенного набора для Telegram Rich Messages.

> Важно: эти функции описаны как примерный набор helper-ов. В текущем коде проекта rich-фото товара уже собираются через `tg://photo?id=...` и `InputRichMessage.media`, потому что Telegram Rich Message API корректно принимает медиа именно через явную привязку media.

---

## 1. `html_escape(value)`

### Вход

```python
html_escape('<b>Tom & "Jerry"</b>')
```

### Вывод

```html
&lt;b&gt;Tom &amp; &quot;Jerry&quot;&lt;/b&gt;
```

### Что делает

Экранирует потенциально опасный HTML:

- `<` превращает в `&lt;`;
- `>` превращает в `&gt;`;
- `&` превращает в `&amp;`;
- `"` превращает в `&quot;`.

То есть пользовательский текст не ломает rich-разметку и не может вставить собственный HTML.

---

## 2. `build_rich_message(html)`

### Вход

```python
build_rich_message('<h2>Пример</h2>').model_dump()
```

### Вывод

```python
{
    'html': '<h2>Пример</h2>',
    'skip_entity_detection': True,
}
```

### Что делает

Оборачивает HTML в объект `InputRichMessage`.

Вместо старого варианта:

```python
message.answer(text, parse_mode="HTML")
```

используется rich-вариант:

```python
message.answer_rich(
    rich_message=InputRichMessage(
        html="<h2>Пример</h2>",
        skip_entity_detection=True,
    )
)
```

## 3. `build_table_html(headers, rows)`

### Вход

```python
build_table_html(
    ("Name", "City", "Age"),
    (
        ("Anna", "Moscow", 25),
        ("Ivan <admin>", "Kazan & Tatarstan", 31),
    ),
)
```

### Вывод

```html
<table bordered striped>
  <tr><td><strong>Name</strong></td><td><strong>City</strong></td><td><strong>Age</strong></td></tr>
  <tr><td>Anna</td><td>Moscow</td><td>25</td></tr>
  <tr><td>Ivan &lt;admin&gt;</td><td>Kazan &amp; Tatarstan</td><td>31</td></tr>
</table>
```

### Что делает

Создаёт rich-таблицу:

- `bordered` — таблица с границами;
- `striped` — таблица с чередованием строк;
- заголовки рендерятся жирными через `<strong>`;
- значения ячеек экранируются через `html_escape`.

Например:

```html
Ivan <admin>
```

становится:

```html
Ivan &lt;admin&gt;
```

А:

```html
Kazan & Tatarstan
```

становится:

```html
Kazan &amp; Tatarstan
```

## 4. `build_tariffs_table_html()`

### Вход

```python
build_tariffs_table_html()
```

### Вывод

```html
<table bordered striped>
  <tr><td><strong>Тариф</strong></td><td><strong>Цена</strong></td><td><strong>Лимит</strong></td></tr>
  <tr><td>Start</td><td>299₽</td><td>100 запросов</td></tr>
  <tr><td>Pro</td><td>799₽</td><td>500 запросов</td></tr>
</table>
```

### Что делает

Генерирует готовую таблицу тарифов.

В Telegram это будет выглядеть примерно как аккуратная таблица:

| Тариф | Цена | Лимит |
|---|---:|---|
| Start | 299₽ | 100 запросов |
| Pro | 799₽ | 500 запросов |

Для rental-бота аналогично можно сделать таблицу тарифов аренды:

```html
<table bordered striped>
  <tr><td><strong>Тариф</strong></td><td><strong>Цена</strong></td><td><strong>Лимит</strong></td></tr>
  <tr><td>Мини</td><td>1 100 ₽ / день</td><td>1 день</td></tr>
  <tr><td>Профи</td><td>7 000 ₽ / неделя</td><td>7 дней</td></tr>
</table>
```

## 5. `build_images_slideshow_html(images)`

### Вход

```python
build_images_slideshow_html(
    (
        "https://example.com/1.jpg",
        "https://example.com/2.jpg",
    )
)
```

### Вывод

```html
<tg-slideshow>
  <img src="https://example.com/1.jpg">
  <img src="https://example.com/2.jpg">
</tg-slideshow>
```

### Что делает

Создаёт rich-карусель / slideshow.

Подходит для:

- фото товара;
- результатов генерации;
- онбординга;
- карточек “до/после”;
- галереи услуги.

В рабочем коде проекта для реального Telegram rich-media лучше использовать не прямые URL в `src`, а связку:

```html
<img src="tg://photo?id=item_photo_0">
```

и отдельно передавать media через `InputRichMessage.media`.

## 6. `build_images_collage_html(images)`

### Вход

```python
build_images_collage_html(
    (
        "https://example.com/1.jpg",
        "https://example.com/2.jpg",
    )
)
```

### Вывод

```html
<tg-collage>
  <img src="https://example.com/1.jpg">
  <img src="https://example.com/2.jpg">
</tg-collage>
```

### Что делает

Создаёт rich-коллаж.

### Разница между `tg-slideshow` и `tg-collage`

#### `tg-slideshow`

Показывает одно фото или блок за раз, пользователь листает изображения внутри Telegram-клиента.

Лучше подходит для:

- карточки товара;
- галереи товара;
- фото техники;
- результатов “вариант 1 / вариант 2 / вариант 3”.

#### `tg-collage`

Показывает несколько фото сразу сеткой.

Лучше подходит для:

- витрины;
- сравнения;
- “до/после”;
- блока “примеры работ”;
- мини-галереи без необходимости листать.

Для карточки товара лучше использовать `tg-slideshow`, потому что он заменяет старую фото-карусель.

## 7. `build_user_stats_html(user)`

### Вход

```python
build_user_stats_html(
    {
        "full_name": "Иван <Иванов>",
        "city": "Казань",
        "rentals_count": 7,
        "rating": 4.9,
    }
)
```

### Вывод

```html
<h2>Профиль</h2>
<p>Краткая статистика пользователя.</p>
<table bordered striped>
  <tr><td><strong>Показатель</strong></td><td><strong>Значение</strong></td></tr>
  <tr><td>Имя</td><td>Иван &lt;Иванов&gt;</td></tr>
  <tr><td>Город</td><td>Казань</td></tr>
  <tr><td>Аренды</td><td>7</td></tr>
  <tr><td>Рейтинг</td><td>4.9</td></tr>
</table>
```

### Что делает

Генерирует rich-блок профиля пользователя:

- заголовок `<h2>`;
- абзац `<p>`;
- таблицу статистики.

Имя пользователя экранируется:

```html
Иван <Иванов>
```

становится:

```html
Иван &lt;Иванов&gt;
```

## 8. `build_details_html(summary, body_html)`

### Вход

```python
build_details_html(
    "Показать подробности",
    "<p>Тут длинное описание...</p>",

```

### Вывод

```html
<details>
  <summary>Показать подробности</summary>
  <p>Тут длинное описание...</p>
</details>
```

### Что делает

Создаёт раскрывающийся rich-блок.

Подходит для:

- длинного описания товара;
- условий аренды;
- правил доставки;
- “что входит в комплект”;
- предупреждений;
- FAQ.

Пример для товара:

```html
<details>
  <summary>Показать описание</summary>
  <p>Виброплита подходит для уплотнения песка, щебня и грунта.</p>
</details>
```

# Мини-вывод по полезности для rental-бота

## Стоит использовать сразу

### `html_escape`

Обязательно. Это база безопасности для rich HTML.

### `build_table_html`

Очень полезно. На нём можно собрать:

- карточку товара;
- характеристики;
- тарифы;
- платежи;
- статистику;
- админские отчёты.

### `build_details_html`

Очень полезно для длинных описаний товара.

### `build_images_slideshow_html`

Полезно концептуально, но в рабочем коде проекта лучше использовать текущую схему с `tg://photo?id=...` + `InputRichMessage.media`, потому что так Telegram нормально принимает `file_id`.

## Можно использовать позже

### `build_tariffs_table_html`

Хорошо для раздела тарифов/пакетов.

### `build_user_stats_html`

Хорошо для профиля и админки.

### `build_images_collage_html`

Полезно для витрин/подборок, но не как замена фото-карусели товара. Для товара лучше `tg-slideshow`.
