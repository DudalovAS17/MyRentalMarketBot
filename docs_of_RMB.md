# RentalMarketBot — полное описание проекта

## 1. Главная идея бота

**RentalMarketBot** — это Telegram-бот для компании по аренде строительного оборудования, инструментов и техники.

Это не маркетплейс, где пользователи сами создают объявления. Это **бот-каталог компании**, где клиент выбирает оборудование, смотрит карточки товаров, оставляет заявку, а менеджер обрабатывает её в админке.

Главная бизнес-задача:

```text
клиенту удобно выбрать оборудование
→ быстро оставить заявку
→ менеджер сразу получает структурированную заявку
→ компания не теряет обращения из Telegram
```

Бот — это не замена сайта 1 в 1. Это **Telegram-канал заявок + каталог + менеджерская система**.

---

## 2. Для кого этот бот

В первую очередь бот подходит для компаний в нишах:

```text
аренда строительного оборудования
аренда электроинструмента
аренда генераторов
аренда садовой техники
аренда спецтехники
прокат оборудования для ремонта/стройки
```

У такой компании уже есть:

```text
товары
цены
фото
категории
менеджеры
сайт или каталог
```

Ей нужен не “бот с объявлениями”, а **Telegram-витрина + обработка заявок**.

---

## 3. Основная логика продукта

В новом проекте логика такая:

```text
Компания владеет товарами.
Админ/менеджер управляет каталогом.
Пользователь смотрит каталог и оставляет заявку.
```

Переосмысление сущностей:

```text
User ≠ арендодатель
User = клиент

Item ≠ объявление пользователя
Item = товар каталога компании

Rental ≠ сделка между двумя пользователями
Rental = заявка клиента на аренду

Admin ≠ просто пользователь с флагом is_admin
Admin = менеджер/сотрудник компании
```

## 4. Главный пользовательский сценарий

Основной путь клиента:

```text
/start
→ Главное меню
→ Каталог аренды
→ Категория
→ Подкатегория
→ Карточка товара
→ Листание товаров стрелками
→ Оставить заявку
→ Срок аренды
→ Количество
→ Доставка
→ Контакты
→ Комментарий
→ Заявка отправлена менеджеру
```

Пример:

```text
/start
→ 🏗 Каталог аренды
→ Дорожно-строительная техника
→ Виброплиты
→ Виброплита Champion PC9045F
→ 🛒 Оставить заявку
```

## 5. Главное меню пользователя

Базовое меню пользователя:

```text
🏗 Каталог аренды
🔎 Поиск оборудования
📋 Мои заявки
ℹ️ Условия аренды
🚚 Доставка и оплата
💬 Поддержка
📞 Связаться с менеджером
```

Позже можно добавить:

```text

⭐ Популярное
🔥 Рекомендуем
❤️ Избранное
```

Для MVP это не обязательно.

## 6. Каталог

Каталог строится по структуре:

```text
Категории
→ Подкатегории
→ Товары
```

Для первой версии зафиксированы 4 категории:

```text
1. Дорожно-строительная техника
2. Электро инструменты
3. Генераторы
4. Садовая техника
```

Пример подкатегорий:

```text
Дорожно-строительная техника
→ Виброплиты
→ Вибротрамбовки
→ Резчики швов
→ Бензорезы
```

Эта структура хорошо ложится на Telegram-меню.

---

## 7. Просмотр товаров

Главное UX-решение: **не показывать товары простым списком**.

Вместо списка используется **карусель карточек**.

Пользователь заходит в подкатегорию:

```text
Виброплиты
```

И видит первый товар:

```text
[Фото товара]

Виброплита Champion PC9045F

Цена: от 1100 ₽/сутки
Глубина уплотнения: до 200 мм
Вес: 91 кг
Мощность: 5,5 л.с.

Товар 1 из 9
```

Кнопки:

```text
⬅️ 1/9 ➡️
🛒 Оставить заявку
ℹ️ Подробнее
💬 Задать вопрос
⬅️ Назад
```

Это делает обычного Telegram-бота похожим на нормальный каталог.

## 8. Карточка товара

Карточка товара должна быть сильной, почти как мини-версия страницы сайта.

В ней должны быть:

```text
название
главное фото
цена
краткое описание
характеристики
количество/наличие
статус товара
кнопка заявки
кнопка вопроса
переключение фото
```

Пример карточки:

```text
🟡 Виброплита Champion PC9045F

💰 Аренда: от 1100 ₽/сутки
✅ Без залога

📌 Подходит для:
земляных работ, дома и дачи, бетонных работ

Характеристики:
• Глубина уплотнения: до 200 мм
• Вес: 91 кг
• Мощность: 5,5 л.с.
• Двигатель: Honda GX160

📦 В наличии: 2 шт.
```

Кнопки:

```text
⬅️ Товар | 1/9 | ➡️
🖼 Фото | 1/6 | ➡️
🛒 Оставить заявку
💬 Задать вопрос
⬅️ В подкатегорию
```

## 9. Заявка на аренду

В MVP не нужно сразу делать оплату внутри бота.

Для аренды строительного оборудования реалистичнее сценарий:

```text
клиент оставляет заявку
→ менеджер уточняет наличие, дату, доставку, цену
→ заявка подтверждается вручную
```

Форма заявки:

```text
1. Какой товар?
2. Сколько штук?
3. На какой срок?
4. Нужна ли доставка?
5. Адрес доставки, если нужна
6. Имя
7. Телефон
8. Комментарий
```

Итоговая заявка админу:

```text
🆕 Новая заявка #42

Товар: Виброплита Champion PC9045F
Количество: 1
Срок: 2–3 суток
Доставка: нужна
Адрес: ...
Клиент: Алексей
Телефон: +7...
Комментарий: нужна на выходные
```

Кнопки менеджера:

```text
✅ Взять в работу
✅ Подтвердить
❌ Отклонить
📞 Связаться
📝 Комментарий
```

---

## 10. Статусы заявки

Базовые статусы заявки:

```text
requested / new — новая заявка
in_progress — в работе
confirmed — подтверждена
rejected — отклонена
cancelled — отменена
completed — завершена
```

В текущем коде модель может называться `Rental`, таблица `rentals`, но по смыслу это уже **заявка клиента на аренду**.

Позже можно концептуально переименовать:

```text
RentalRequest
rental_requests
```

Но на текущей стадии можно продолжать с `Rental`, понимая, что это заявка.

## 11. Админка

Админка — важнейшая часть проекта.

Для компании бот полезен не только каталогом, но и тем, что менеджеру удобно работать.

Главное меню админа:

```text
👑 Админ-панель

📦 Каталог товаров
➕ Добавить товар
🆕 Новые заявки
📋 Все заявки
💬 Поддержка
👥 Клиенты
📊 Статистика
⚙️ Настройки
```

---

## 12. Админское управление товарами

То, что раньше было “создать объявление”, теперь становится админским сценарием:

```text
➕ Добавить товар
📦 Каталог товаров
✏️ Редактировать товар
🗑 Скрыть / архивировать товар
📷 Добавить фото товара
💰 Изменить цену
✅ Опубликовать товар
```

Старая логика создания объявлений не выбрасывается, а переносится в админку.

Пользователь не может создавать товар.  
Админ может.

---

## 13. Роли админов

Отдельная модель `Admin` нужна для менеджеров и сотрудников компании.

Роли:

```text
owner — владелец системы
admin — администратор каталога
manager — менеджер заявок
```

Пример разграничения:

```text
owner:
полный доступ

admin:
товары, категории, заявки, настройки, поддержка

manager:
заявки и поддержка
```

---

## 14. Логи действий админа

`AdminAction` фиксирует действия в админке.

Она нужна, чтобы понимать:

```text
кто создал товар
кто изменил цену
кто скрыл товар
кто подтвердил заявку
кто отклонил заявку
кто закрыл тикет
```

Пример записи:

```text
admin_id: 1
action_type: update
entity_type: item
entity_id: 12
note: Изменил цену аренды
payload: {"old_price": "1100", "new_price": "1200"}
```

## 15. Поддержка

Поддержка нужна обязательно.

Пользователь может:

```text
задать вопрос по товару
спросить про доставку
спросить про оплату
уточнить заявку
написать менеджеру
```

Сценарий:

```text
💬 Поддержка
→ выбрать тему
→ написать сообщение
→ тикет уходит админу
```

Или из карточки товара:

```text
💬 Задать вопрос по товару
```

Тогда тикет можно связать с товаром/заявкой.

---

## 16. Отзывы

Отзывы не обязательны для MVP, но хороши для портфолио и будущей коммерческой версии.

Новая логика отзывов:

```text
клиент оставляет отзыв не другому пользователю,
а о товаре / заявке / сервисе компании.
```

Сценарий:

```text
после завершения аренды
→ бот просит оценить сервис
→ клиент ставит 1–5
→ пишет комментарий
→ отзыв попадает на модерацию
```

---

## 17. Фото и галерея

Фото — одна из главных частей каталога.

У товара должно быть:

```text
одно главное фото
несколько дополнительных фото
порядок отображения
telegram_file_id
url
```

`url` важен, потому что можно импортировать фото с сайта компании.

`telegram_file_id` важен, потому что бот сможет быстро отправлять фото без повторной загрузки.

---

## 18. Характеристики товара

`ItemCharacteristic` позволяет не зашивать характеристики в описание, а хранить их структурно.

Пример:

```text
Глубина уплотнения → до 200 мм
Вес → 91 кг
Мощность → 5,5 л.с.
Двигатель → Honda GX160
```

Это нужно для красивой карточки товара.

---

## 19. Корзина и несколько товаров в заявке

Для MVP можно сделать проще:

```text
одна заявка = один товар
```

Но в будущем нужна более взрослая логика:

```text
корзина
→ несколько товаров
→ одна заявка
```

В ЧНС уже зафиксировано:

```text
1. Создать модели: rental_items и cart_items
```

`cart_items` — что пользователь добавил в корзину.

`rental_items` — какие товары вошли в оформленную заявку.

Это следующий уровень.

## 20. Telegram-канал как воронка

Бот может работать вместе с Telegram-каналом.

Связка:

```text
Telegram-канал
→ полезный пост
→ кнопка “Смотреть в боте”
→ бот открывает нужную подкатегорию/товар
→ клиент оставляет заявку
```

Пример:

```text
Пост: Какую виброплиту выбрать для укладки плитки?
Кнопка: Смотреть виброплиты
Бот: открывает карусель виброплит
```

Так продукт можно продавать не только как “бот”, а как:

```text
Telegram-воронка для аренды оборудования
```

## 23. Что можно продавать клиенту

Формулировка продукта:

> **Telegram-бот-каталог для аренды строительного оборудования. Клиент выбирает оборудование в Telegram, смотрит фото, характеристики и цену, оставляет заявку, а менеджер получает её в структурированном виде и обрабатывает через админ-панель.**

Более сильная формулировка:

> **Telegram-система заявок для компании по аренде оборудования: каталог, карточки товаров, заявки, менеджерская админка, поддержка и уведомления.**

## 24. Чем этот бот отличается от сайта

Сайт нужен для поиска из Яндекса/Google и красивого каталога.

Бот нужен для:

```text
быстрого контакта
заявок из Telegram
уведомлений
возврата клиента
диалога с менеджером
рассылок
канала продаж
поддержки
```

Бот не убивает сайт. Он дополняет сайт.

---

## 25. Чем этот бот отличается от Mini App

Mini App — это красивая витрина.

Но обычный бот уже может дать:

```text
каталог
карточки с фото
карусель товаров
заявки
админку
поддержку
уведомления
```

Поэтому для портфолио сначала делаем обычного бота.

Следующая версия может быть:

```text
Mini App-каталог с сеткой товаров и фильтрами
```

## 26. Итоговый продукт

# RentalMarketBot

**Telegram-система аренды оборудования для компании.**

```text
1. Клиентская часть
   каталог, товары, карточки, заявки, поддержка

2. Админская часть
```

```text

```
